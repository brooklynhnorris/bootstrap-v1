OPQ-001 | Core Page Zero Body Text Emergency
Trigger Source: page_crawl_snapshots
Trigger Condition: ```sql
SELECT
  url,
  word_count,
  page_type,
  has_central_entity,
  schema_types,
  is_noindex
FROM page_crawl_snapshots
WHERE page_type = 'core'
  AND word_count = 0
  AND is_noindex = FALSE
```
Threshold: word_count = 0 (any indexed core page with no body text fires immediately)
Data Fields:
Needs New Data: None. All fields (`word_count`, `page_type`, `is_noindex`) are standard crawl outputs.
Investigation Steps:

Diagnosis: An indexed core page with zero body text is invisible to both crawlers and AI citation systems     MSE elements alone cannot carry a page that has no parseable text whatsoever. This is a critical content void, not a style choice.

**ACTION_OUTPUT (Play-Brief Format):**
- **Current state    ** Indexed core page has 0 body words; crawlers find no text signal; AI systems have no extractable content to cite; product context (Z-Frame construction, SafeTack configuration, SafeBump roof specification) is completely absent.
- **Your move    ** Add a minimum 50-word, maximum 500-word body block. Open with a first sentence that names the trailer model and its defining system (e.g., "The [Model Name] is built on Double D Trailers' Z-Frame     high-tensile, zinc-infused construction welded into a single torsion unit"). Follow the first-sentence-answers-heading pattern. Place one SafeTack, SafeBump, or SafeKick term in body if applicable to the model.
- **Done when    ** `word_count` is between 50 and 500 AND `has_central_entity = TRUE` on next crawl snapshot.
- **Recheck    ** 48 hours post-publish crawl confirmation; re-run this query weekly as a standing zero-content sentinel.
Action Output:
Priority: Critical
Assigned:
AI Search Relevance: AI citation systems (Google AI Overviews, Perplexity, ChatGPT Browse) require extractable prose to quote, paraphrase, or cite. A zero-text core page cannot be cited for any query variant of "custom horse trailers," "safest horse trailer construction," or "Z-Frame trailer." Even one well-formed sentence containing a named entity (Z-Frame, SafeTack) makes the page citable. This rule prevents complete AI invisibility.

OPQ-005 | H1-to-Title-Tag Mismatch Detector
Trigger Source: page_crawl_snapshots
Trigger Condition: ```sql
SELECT
  url,
  h1,
  title_tag,
  page_type,
  h1_matches_title
FROM page_crawl_snapshots
WHERE h1_matches_title = FALSE
  AND is_noindex = FALSE
  AND is_utility = FALSE
  AND page_type IN ('core', 'outer')
```
Threshold: `h1_matches_title = FALSE`     H1 and title tag are not topically aligned (exact match not required; semantic alignment required)
Data Fields:
Needs New Data: `h1_matches_title`     requires semantic similarity scoring (not exact string match) in the crawl pipeline. A cosine similarity threshold of    0.75 between H1 and title tag text is recommended as the pass condition. If only exact-match boolean is available, upgrade to semantic scoring before relying on this rule at scale.
Investigation Steps:

Diagnosis: When the H1 and title tag describe different topics or use contradictory entity language, crawlers and AI systems receive conflicting signals about the page's subject     creating ranking ambiguity and citation ineligibility for either signal.

**ACTION_OUTPUT (Play-Brief Format):**
- **Current state    ** H1 and title tag are misaligned; the page sends two different topical signals; AI systems cannot determine the canonical subject of the page for citation purposes; traditional ranking is split between two keyword intents.
- **Your move    ** Align the H1 and title tag to the same central entity and primary predicate. They do not need to be identical     the title tag may include the brand name suffix ("| Double D Trailers") and can be slightly shorter     but both must name the same core subject. Example correct alignment: Title: "[Model Name] Custom Horse Trailer | Double D Trailers" / H1: "[Model Name] Horse Trailer with Z-Frame Construction."
- **Done when    ** `h1_matches_title = TRUE` on next crawl (semantic match, not character-for-character identity).
- **Recheck    ** Every crawl cycle; flag immediately after any CMS title or H1 edit event.
Action Output:
Priority: High
Assigned:
AI Search Relevance: AI systems extract page subject from a combination of title tag, H1, and opening sentence. When title tag says "Custom Horse Trailer" and H1 says "Horse Trailer Safety Features," the system cannot resolve a single subject for citation. Aligning both to the same entity (e.g., "[Model Name] with SafeTack Reverse") gives AI systems a consistent signal across all three extraction points, maximizing citation precision for model-specific queries.

OPQ-006 | Duplicate H1 and Title Tag Cross-Page Collision Detector
Trigger Source: page_crawl_snapshots
Trigger Condition: ```sql
-- Duplicate H1s across indexed pages
SELECT url, h1, title_tag, page_type,
       COUNT(*) OVER (PARTITION BY h1) AS h1_duplicate_count
FROM page_crawl_snapshots
WHERE is_noindex = FALSE
  AND is_utility = FALSE
  AND h1 IS NOT NULL

UNION ALL

-- Duplicate title tags across indexed pages
SELECT url, h1, title_tag, page_type,
       COUNT(*) OVER (PARTITION BY title_tag) AS title_duplicate_count
FROM page_crawl_snapshots
WHERE is_noindex = FALSE
  AND is_utility = FALSE
  AND title_tag IS NOT NULL

ORDER BY url
```
Threshold: Any H1 or title tag appearing on more than one indexed, non-utility page
Data Fields:
Needs New Data: None. Standard crawl deduplication query.
Investigation Steps:

Diagnosis: Duplicate H1s or title tags across pages cause crawlers to treat those pages as competing for the same query     splitting ranking authority     and cause AI systems to arbitrarily choose one page as the citation source while suppressing the other.

**ACTION_OUTPUT (Play-Brief Format):**
- **Current state    ** Two or more indexed pages share an identical H1 or title tag; Google cannot determine the canonical ranking target; AI systems select one page semi-randomly for citation and ignore the duplicate.
- **Your move    ** Differentiate each page's H1 and title tag by its specific entity and differentiating predicate. For trailer model pages: include the specific model configuration. For outer pages: include the specific angle (e.g., "How Z-Frame Construction Reduces Trailer Flex" vs. "Z-Frame vs. Conventional Welding in Horse Trailers"). Never use a generic H1 like "Horse Trailer Features" on more than one page.
- **Done when    ** Each indexed page has a unique H1 AND a unique title tag confirmed on next crawl.
- **Recheck    ** Run at every crawl cycle as a standing uniqueness check; alert immediately when a new page is published with an H1 or title already in use.
Action Output:
Priority: High
Assigned:
AI Search Relevance: AI citation systems build an internal index of page subjects. When two pages share an H1, they appear interchangeable     the AI cites whichever it encounters first and treats the other as redundant. Unique, entity-specific H1s ensure each Double D Trailers page (e.g., a SafeTack page vs. a SafeBump page) occupies a distinct citation slot in AI-generated answers.

Trigger Source: `page_crawl_snapshots`
Trigger Condition: ```sql

FROM page_crawl_snapshots
WHERE page_type = 'core'
  AND is_noindex = TRUE
  AND http_status = 200
```
Threshold: Any single instance. Zero tolerance.
Data Fields:
Needs New Data: None     `is_noindex`, `page_type`, `http_status` already in `page_crawl_snapshots`
Investigation Steps:

Diagnosis: A core product page returning HTTP 200 with `is_noindex = TRUE` is actively instructing every crawler     Googlebot, GPTBot, PerplexityBot, ClaudeBot     to exclude it from all indexes. No indexing means no ranking and no AI citation, regardless of content quality or MSE completeness.
Action Output: - **Current state    ** Core URL is live but carrying a noindex directive; invisible to all search and AI systems

- **Verify    ** Re-crawl flagged URL and confirm `is_noindex = FALSE`; submit URL in Google Search Console for immediate reindexing
- **Done when    ** `is_noindex = FALSE` on all core pages with `http_status = 200`
- **Recheck    ** Every crawl cycle; noindex can be reintroduced by CMS template changes or plugin updates
Priority:      CRITICAL
Assigned:
AI Search Relevance: GPTBot, PerplexityBot, and ClaudeBot respect noindex directives. A noindexed page featuring Z-Frame construction details, SafeTack reverse-load geometry, or SafeBump roof specifications cannot be retrieved, read, or cited by any AI search system. Removing noindex is the prerequisite for all AI citation eligibility.

USE-R2 | Engagement Decay Detection on Outer Pages     CTR Collapse Without Position Loss
Trigger Source: `gsc_snapshots` (dual-period)
Trigger Condition: ```sql
-- Primary: Absolute CTR floor on high-impression outer pages
SELECT
  page,
  query,
  impressions,
  clicks,
  ctr,
  position
FROM gsc_snapshots
WHERE page_type = 'outer'
  AND impressions >= 1000
  AND position BETWEEN 1.0 AND 20.0
  AND ctr < 0.01
ORDER BY impressions DESC
LIMIT 30;

-- Secondary: CTR decay with stable or improved position
SELECT
  curr.page,
  curr.query,
  curr.ctr          AS ctr_current,
  prev.ctr          AS ctr_prior,
  (curr.ctr - prev.ctr) AS ctr_delta,
  curr.position     AS pos_current,
  prev.position     AS pos_prior
FROM gsc_snapshots curr
JOIN gsc_snapshots prev
  ON curr.page  = prev.page
  AND curr.query = prev.query
  AND curr.date_range = '28d_current'
  AND prev.date_range = '28d_prior'
WHERE curr.page_type = 'outer'
  AND (curr.ctr - prev.ctr) < -0.005
  AND curr.position <= prev.position
ORDER BY ctr_delta ASC;
```
Threshold: Primary: CTR < 1% on outer pages with    1,000 impressions at P1   20 | Secondary: CTR decline > 0.5 percentage points period-over-period with position held or improved
Data Fields:
Needs New Data: Secondary query requires two stored `date_range` partitions (`28d_current`, `28d_prior`) in `gsc_snapshots`     confirm partitioned snapshots exist before scheduling secondary run
Investigation Steps:

Diagnosis: A secondary-flag outer page (position held, CTR dropped) is experiencing SERP feature displacement, meaning a new AI Overview, featured snippet, or rich result is absorbing clicks that previously reached the organic listing. This is distinct from a ranking problem and requires a content format response, not a ranking intervention.
Action Output: - **Current state    ** Primary flags indicate absolute CTR failure; secondary flags indicate displacement     triage separately; prioritize outer pages tied to Z-Frame material properties, SafeBump roof engineering, or horse trailer buyer decision queries
- **Your move (primary)    ** Audit meta title/description for query specificity gap; rewrite to surface the specific informational answer the query implies; confirm `word_count >= 1000` in `page_crawl_snapshots`     sub-1000-word outer pages are expanded under USE-R4
- **Your move (secondary/displacement)    ** Add FAQ schema targeting the displacing featured snippet query; restructure relevant H2 as a direct question answered in the first sentence of that section; add a definition-style answer block under the H2 to re-enter AI snippet extraction candidacy
- **Done when    ** Updated snippet deployed; schema validated in GSC Rich Results Test
- **Recheck    ** 28-day CTR delta for same query-page pairs; secondary: monitor for featured snippet acquisition in GSC Search Appearance filter

---
Priority:      High
Assigned:
AI Search Relevance: Outer pages with collapsing CTR while holding position indicate SERP feature displacement (AI Overviews, featured snippets, image packs absorbing clicks above the organic result). Pages that lose AI snippet eligibility lose citation eligibility simultaneously. Restoring structured content formats (FAQ schema, definition-style H2s, numbered sequences) directly re-enters these pages into AI extraction candidate pools.
